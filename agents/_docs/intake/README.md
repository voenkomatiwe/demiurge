# Intake — Raw Project Materials

Drop any files here that describe your project idea. PM will read everything and generate a structured Brief for your approval.

## What to put here

- Text notes (`.md`, `.txt`) — product ideas, feature descriptions, business context
- Screenshots and mockups (`.png`, `.jpg`, `.pdf`) — reference designs, competitor examples
- Links and references (in a `.md` file) — competitor URLs, articles, inspiration
- Existing specs or docs — anything that gives PM context about what you want to build

## How it works

1. **You** drop files into this folder
2. **You** tell PM: `read docs/intake/ and generate a brief`
3. **PM** reads everything, synthesizes a Brief (`docs/BRIEF.md`)
4. **You** review and approve the Brief (or ask PM to adjust)
5. **PM** creates `TASK-001.md` from the approved Brief
6. Files stay here as an archive — PM can re-read them if context is needed later

## Tips

- More context = better Brief. Don't hold back.
- If you have a clear vision, a single `.md` with bullet points is enough.
- If you're exploring, screenshots of competitors + a paragraph of "what I like about this" works great.
- PM will ask clarifying questions if something is ambiguous.

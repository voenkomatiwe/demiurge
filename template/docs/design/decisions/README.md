## When to write a design ADR

- Choosing between two component patterns with long-term implications (e.g., navigation style)
- Changing an established token (e.g., colour rebrand, type scale migration)
- Accessibility trade-offs that can't satisfy every principle
- Adopting or rejecting a third-party component library

Small day-to-day styling choices don't need ADRs — just document them in `system.md` under the relevant component.
